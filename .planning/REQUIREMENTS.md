# Requirements: Developer portfolio (static)

**Defined:** 2026-04-27  
**Core Value:** Visitors can quickly understand who you are as a developer, see representative work, and contact you from one fast, readable page.

## v1 Requirements

### Experience & content

- [ ] **EXP-01**: Single page presents hero with title, value statement, and CTAs (projects, contact)
- [ ] **EXP-02**: About section communicates professional mindset (learning, problem-solving, building useful software) without being overly personal
- [ ] **EXP-03**: Featured projects section shows three static cards (desktop, web, research categories) with placeholders for GitHub and demo links
- [ ] **EXP-04**: Strengths section emphasizes capabilities (not tool hype)
- [ ] **EXP-05**: Tools section is compact and secondary
- [ ] **EXP-06**: Contact section lists email, GitHub, LinkedIn placeholders and a short closing line

### Interaction & layout

- [ ] **UX-01**: Smooth scrolling to in-page sections from navigation
- [ ] **UX-02**: Responsive layout for desktop and mobile
- [ ] **UX-03**: Mobile-friendly navigation (menu when needed)

### Technical

- [ ] **TEC-01**: Project builds with Vite + React + TypeScript + Tailwind
- [ ] **TEC-02**: No backend, database, CMS, or external API calls at runtime
- [ ] **TEC-03**: Static data arrays for projects, strengths, and links for easy editing
- [ ] **TEC-04**: README documents local setup and GitHub Pages deployment including `base` path notes

## v2 Requirements

### Enhancements (deferred)

- **CI-01**: GitHub Actions workflow for automatic Pages deploy
- **ANA-01**: Privacy-friendly analytics (optional)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Server-side rendering / API routes | Static-only deployment target |
| Authentication or admin UI | No backend; content edited in repo |
| Blog or multi-page IA | Explicitly single-page portfolio |
